## 2. Среда и воспроизводимость

- Устройство (CPU/GPU): CPU

## 3. Данные

- Датасет: EMNIST Balanced
- Разделение: train/val/test train split 80/20 + test из torchvision
- Трансформации (transform): ToTensor / Normalize 
- Комментарий (1-3 предложения): число классов - 47; размерность входных данных - (1,28,28); использовался FAST_MODE для экономии времени, поэтому количество батчей получилось не таким большим

## 4. Базовая модель и обучение

- Модель MLP (кратко): 2 скрытых слоя, 256 и 128, активация через ReLu
- Loss: CrossEntropyLoss
- Базовый Optimizer (для части A): Adam (lr=1e-3)
- Batch size: x = (128, 1, 28, 28); x_flat = (128, 784)
- Epochs (макс): 15
- EarlyStopping: (patience=4, metric=val_accuracy)

## 5. Часть A (S08): регуляризация (E1-E4)

Опишите, что меняли. Формулировки должны быть короткими и сопоставимыми.

- E1 (base): 2 скрытых слоя, без Dropout/BatchNorm
- E2 (Dropout): как E1 + Dropout(p=0.3)
- E3 (BatchNorm): как E1 + BatchNorm
- E4 (EarlyStopping): лучший из (E2/E3) (в нашем случае E3) + EarlyStopping

## 6. Часть B (S09): LR, оптимизаторы, weight decay (O1-O3)

- O1: LR слишком большой (Adam, lr=0.5)
- O2: LR слишком маленький (Adam, lr=1e-5)
- O3: SGD+momentum (momentum=0.9) + weight_decay=1e-5 (lr=1e-3)
